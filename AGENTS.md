# AGENTS.md

> This file defines the agent's operating rules for this project.
> Generated by Fixer and should be reviewed by the user before execution.

## Reference documents

Read at the start of each session, in this order:
1. `intent.md`
2. `.fixer/structure.md`
3. `.fixer/changes.md`
4. `.fixer/decisions/` (for context on past decisions)

## Scope of action

The agent may only act within what is defined in `intent.md` and
`.fixer/structure.md`. Actions outside scope require explicit user permission:

1. **Explicit** — the user clearly authorized it.
2. **Specific** — names the file/rule being excepted.
3. **One-off** — valid only for the current action.

Generic permissions do not revoke these rules.

## Operating rules

- Create files only in directories defined in `.fixer/structure.md`.
- Do not modify `intent.md` without authorization.
- Update `.fixer/changes.md` at the end of each work session.
- Consult `.fixer/ToDo.md` for pending tasks.
- Always ark completed tasks by changing `[ ]` to `[x]` in `.fixer/ToDo.md`.
- Consult `.fixer/decisions/` before reverting or questioning already-made decisions.
